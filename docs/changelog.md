# Changelog

## 2026-03-13
- Added the final beta-pack showcase cases in `data/scenarios`, `data/results`, `data/tanks/test_tank_b.json`, and `data/beta-content.manifest.json`: a Tank B top-down roof entry (`beta_b_ap88_roof_crew_knockout`), a Tank A rear vulnerability case (`beta_a_ap75_rear_engine_damage`), and a Tank A short-vs-long-range AP contrast (`beta_a_ap75_frontal_long_range_no_penetration`). The pack now covers 11 curated scenarios instead of 8.
- Added authored `penetrationLossPer100m` support to the shared shell contract and sim-core. AP shells can now lose a small amount of penetration over the authored scenario distance, which keeps the new long-range showcase meaningful without changing the rest of the prototype architecture.
- Added a lightweight manifest-driven regression command through `packages/sim-core/src/runBetaRegression.ts` and the root `npm run beta:refresh` script. It reads the curated manifest, reruns every beta scenario, refreshes the matching result/debug JSON pairs, and fails clearly if any curated case breaks.
- Upgraded the dev viewer launcher and inspection flow: there is now a `Best Showcase` quick-access strip, a `Reset to default scenario` action, clearer manifest/loading/error states, a bundled beta fallback instead of the old prototype-only sample path, and wording cleanups around playback/manual loading.
- Added a compact `Debug Inspector` in `packages/dev-viewer/src/components/ControlPanel.tsx`. Curated scenarios now load their linked debug JSON and summarize hit context, fuse state, damage totals, fragment-generation notes, and surface-damage markers through expandable sections instead of exposing raw JSON by default.
- Validation completed with `npm run beta:refresh`, `npm run typecheck`, `npm run build`, `npm --workspace @tank-sim/dev-viewer run typecheck`, and `npm --workspace @tank-sim/dev-viewer run build`.
- Remaining unfinished work: the curated pack still lacks a miss showcase and historically researched data, the viewer bundle still triggers Vite's large chunk warning, there is still no semantic automated test suite beyond regeneration, and the damage model remains heuristic rather than physical.
- Reworked `packages/dev-viewer` from a sample/manual file loader into a compact beta-style scenario browser. The viewer now boots from a local curated manifest, auto-loads a default showcase scenario on startup, groups the beta pack into readable categories, and lets the user switch replay cases with one click instead of pairing files manually for normal use.
- Added `data/beta-content.manifest.json` as a lightweight content index over the existing `data/` pack. It records beta launcher copy, category groupings, featured scenarios, and the file links between each curated scenario and its matching tank/result/debug artifacts without changing the underlying JSON pipeline.
- Updated the control panel to include a startup help block, scenario browser, curated tank/shell catalog, and a session section that shows which scenario/tank/result/debug files are currently linked. Manual result JSON loading, manual tank JSON loading, sample loading, playback controls, visibility toggles, x-ray mode, legend, and event timeline were kept intact as fallback/debug tools.
- Updated `packages/dev-viewer/vite.config.ts` so the viewer serves the repository `data/` directory as static public content during dev/build. The production bundle now ships the curated manifest plus the referenced tank, shell, scenario, result, and debug JSON files.
- Validation completed with `npm --workspace @tank-sim/dev-viewer run typecheck` and `npm --workspace @tank-sim/dev-viewer run build`.
- Remaining unfinished work: the browser still has no dedicated debug-report inspector beyond linked filenames, there is no search/filter UI yet, the viewer still ships as one large client bundle, and this is still a local curated beta launcher rather than a polished end-user beta release with onboarding, persistence, or broader content coverage.
